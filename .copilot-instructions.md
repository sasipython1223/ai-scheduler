# 🧠 Copilot Instructions for Module 5.5 - Schedule Engine API

## 🎯 Goal

Build a production-grade REST API endpoint to trigger scheduling logic using Modules 5.2 → 5.4.

## 📂 File Structure Requirements

```
backend/src/
├── routes/schedule-api.ts         # Route registration
├── controllers/schedule-controller.ts  # Request handling
├── validators/schedule-validator.ts     # Input validation
├── services/schedule-service.ts         # Business logic orchestration
├── domain/schedule-engine.ts           # Domain logic wrapper
└── tests/schedule-api.test.ts          # API integration tests
```

## ✅ Implementation Requirements

### Route Layer (`routes/schedule-api.ts`)

- Register `POST /api/schedule/calculate` route
- Use validation middleware
- Delegate to controller
- Keep under 50 LOC

### Controller Layer (`controllers/schedule-controller.ts`)

- Accept sanitized request
- Call `ScheduleService.runScheduleCalculation()`
- Return structured JSON response
- Handle errors with appropriate HTTP status codes
- Keep under 100 LOC

### Validation Layer (`validators/schedule-validator.ts`)

- Validate required task fields: `id`, `duration`, `logicLinks`
- Ensure valid types and formats
- Return 400 on validation failure
- Use TypeScript strict typing
- Keep under 80 LOC

### Service Layer (`services/schedule-service.ts`)

- Orchestrate Modules 5.2 → 5.4 pipeline
- Import and use existing forward/backward pass functions
- Pass validated tasks through all 3 stages
- Return structured output
- Keep under 150 LOC

### Domain Layer (`domain/schedule-engine.ts`)

- Wrap all 3 stages in reusable `runFullSchedulePipeline(tasks)` function
- Clean abstraction over module integration
- Keep under 100 LOC

## 📤 Response Format

### Success Response

```json
{
  "status": "success",
  "data": {
    "tasks": [...],
    "floatAnalysis": {...},
    "flagSummary": {...},
    "processingTimeMs": 123
  }
}
```

### Error Response

```json
{
  "status": "error",
  "message": "Invalid task format",
  "details": ["Task A missing duration"],
  "code": "VALIDATION_ERROR"
}
```

## ⚠️ Error Handling Standards

- 400 for validation errors
- 500 for internal failures
- Include error logging
- Add try/catch in controller
- Provide detailed error messages

## 🧪 Test Requirements

Create comprehensive tests in `schedule-api.test.ts`:

- ✅ Valid task input returns 200
- ❌ Invalid payload returns 400
- ✅ Calculated tasks + float summary returned
- ✅ All pipeline stages called correctly
